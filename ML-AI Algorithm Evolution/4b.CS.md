## Data Structures Core

### Array
- **Definition**: A continuous block of memory where elements are stored sequentially, allowing direct access by index
- **Analogy**: Train seats‚Äîyou can jump to seat 10 instantly (O(1) lookup), but it's hard to add new seats in the middle without rearranging everything (O(n) insert/delete)
- **Video**: [Arrays Data Structure - CS Dojo](https://www.youtube.com/watch?v=QJNwK2uJyGs)

### Linked List
- **Definition**: A collection of nodes where each node contains data and a pointer/reference to the next node in sequence
- **Analogy**: Treasure map‚Äîyou must follow the clues step by step to reach each location; you can't jump directly to the 5th treasure
- **Video**: [Linked Lists in 10 minutes - Reducible](https://www.youtube.com/watch?v=_jQhALI4ujg)

### Stack
- **Definition**: A Last-In-First-Out (LIFO) data structure where elements are added and removed from the same end
- **Analogy**: Plates in a cafeteria‚Äîthe last plate placed on top is the first one taken off
- **Video**: [Stack Data Structure - mycodeschool](https://www.youtube.com/watch?v=F1F2imiOJfk)

### Queue
- **Definition**: A First-In-First-Out (FIFO) data structure where elements are added at the rear and removed from the front
- **Analogy**: DMV line‚Äîthe first person in line is the first person served
- **Video**: [Queue Data Structure - mycodeschool](https://www.youtube.com/watch?v=A3ZUpyrnCbM)

### Tree
- **Definition**: A hierarchical data structure with nodes connected by edges, starting from a root node
- **Analogy**: Family tree‚Äîone ancestor at the top with descendants branching down through generations
- **Video**: [Tree Data Structure - freeCodeCamp](https://www.youtube.com/watch?v=oSWTXtMglKE)

### Hash Table
- **Definition**: A data structure that maps keys to values using a hash function for fast lookup, insertion, and deletion
- **Analogy**: Dictionary‚Äîlook up a word instantly by jumping to the right page, rather than reading the entire book
- **Video**: [Hash Tables and Hash Functions - Computer Science](https://www.youtube.com/watch?v=KyUTuwz_b7Q)

## Tree Variants

### Binary Tree
- **Definition**: A tree where each node has at most 2 children (left and right)
- **Analogy**: Fork in the road that only splits into two paths
- **Video**: [Binary Tree Algorithms - William Fiset](https://www.youtube.com/watch?v=6oL-0TdVy28)

### BST (Binary Search Tree)
- **Definition**: A binary tree where left children are smaller than the parent and right children are larger
- **Analogy**: Organized filing cabinet where everything left of a divider is alphabetically before, everything right is after
- **Video**: [Binary Search Tree - CS Dojo](https://www.youtube.com/watch?v=pYT9F8_LFTM)

### Full Binary Tree
- **Definition**: A binary tree where every node has either 0 or 2 children (no nodes with only 1 child)
- **Analogy**: Complete families‚Äîeveryone either has no kids or exactly 2 kids
- **Video**: [Types of Binary Trees - Jenny's Lectures](https://www.youtube.com/watch?v=qH6yxkw0u78)

### Complete Binary Tree
- **Definition**: A binary tree where all levels are filled except possibly the last, which is filled left to right
- **Analogy**: Theater seating‚Äîfill row 1 completely, then row 2, then row 3 from left to right
- **Video**: [Complete Binary Tree Explained - Back To Back SWE](https://www.youtube.com/watch?v=H5JubkIy_p8)

### Balanced Tree
- **Definition**: A tree where the height of left and right subtrees differ by at most 1, ensuring O(log n) operations
- **Analogy**: Librarian carefully organizing shelves so no section becomes too tall or too short
- **Video**: [AVL Trees & Rotations - Abdul Bari](https://www.youtube.com/watch?v=jDM6_TnYIqE)

### Unbalanced Tree
- **Definition**: A tree where branches have significantly different heights, potentially degrading to linear time complexity
- **Analogy**: Neglected library shelves where books pile up on one side, making searches slow
- **Video**: [Balanced vs Unbalanced Trees - CS50](https://www.youtube.com/watch?v=NcZ2cu7gc-A)

## Graph Variants

### Directed Graph
- **Definition**: A graph where edges have direction, allowing travel from one vertex to another but not necessarily back
- **Analogy**: Twitter follows‚Äîyou can follow someone without them following you back
- **Video**: [Graph Theory Introduction - Reducible](https://www.youtube.com/watch?v=LFKZLXVO-Dg)

### Undirected Graph
- **Definition**: A graph where edges are bidirectional, allowing travel in both directions between connected vertices
- **Analogy**: Facebook friends‚Äîfriendship is mutual; if you're connected, you can reach each other
- **Video**: [Directed vs Undirected Graphs - WilliamFiset](https://www.youtube.com/watch?v=AfYqN3fGapc)

### Spanning Tree
- **Definition**: A subset of a graph's edges that connects all vertices with the minimum number of edges and no cycles
- **Analogy**: Power company connecting houses with minimal wire while reaching everyone
- **Video**: [Minimum Spanning Tree - Abdul Bari](https://www.youtube.com/watch?v=4ZlRH0eK-qQ)

### Adjacency List
- **Definition**: Graph representation where each vertex stores a list of its neighboring vertices
- **Analogy**: Contact list in your phone‚Äîeach person has a list of their friends
- **Video**: [Graph Representations - WilliamFiset](https://www.youtube.com/watch?v=k1wraWzqtvQ)

### Adjacency Matrix
- **Definition**: Graph representation using a 2D grid where matrix[i][j] indicates connection between vertices i and j
- **Analogy**: Spreadsheet showing which cities have direct flights between them
- **Video**: [Adjacency Matrix & List - CS Dojo](https://www.youtube.com/watch?v=k1wraWzqtvQ)

- **Video**: [Big O Notation - CS Dojo](https://www.youtube.com/watch?v=v4cd1O4zkGw)

### Big-Œò (Theta)
- **Definition**: Mathematical notation describing both upper and lower bounds (average-case) with a tight bound
- **Analogy**: Your typical commute time‚Äîwhat you actually experience most days

### Big-Œ© (Omega)

- **Analogy**: Your fastest possible commute route‚Äîideal conditions with no traffic
- **Video**: [Big Omega Notation - Neso Academy](https://www.youtube.com/watch?v=8syQKTdgdzc)

## Common Runtimes

### Constant - O(1)
- **Definition**: Algorithm execution time doesn't change regardless of input size
- **Analogy**: Flipping a light switch‚Äîtakes the same time whether you have 1 or 1,000 lights in the house
- **Video**: [Time Complexity O(1) - Simple Snippets](https://www.youtube.com/watch?v=xZfTCT19GqU)

### Logarithmic - O(log n)
- **Definition**: Algorithm that cuts the problem size in half with each step
- **Analogy**: Guessing a number with "higher/lower" hints‚Äînarrow down from 1-100 in ~7 guesses
- **Video**: [O(log n) Explained - Back To Back SWE](https://www.youtube.com/watch?v=wjDY5RbILno)

### Linear - O(n)
- **Definition**: Algorithm execution time grows proportionally with input size
- **Analogy**: Checking every item on a shopping list‚Äîtwice the items means twice the time
- **Video**: [Linear Time Complexity - Simply Explained](https://www.youtube.com/watch?v=Ee0HzlnIYWQ)

### Polynomial - O(n¬≤)
- **Definition**: Algorithm where execution time grows as the square of the input size
- **Analogy**: Comparing every student with every other student for a group project‚Äîdoubles students = 4x comparisons
- **Video**: [Quadratic Time Complexity - CS Dojo](https://www.youtube.com/watch?v=KatlvCFHPRo)

### Exponential - O(2‚Åø)
- **Definition**: Algorithm where execution time doubles with each additional input element
- **Analogy**: Folding paper in half repeatedly‚Äîby fold 42, it theoretically reaches the moon
- **Video**: [Exponential Time Complexity - Simple Snippets](https://www.youtube.com/watch?v=qH3O_YtM9Ok)

### Factorial - O(n!)
- **Definition**: Algorithm where execution time grows as the factorial of the input size
- **Analogy**: Listing every possible seating arrangement for 20 people at dinner‚Äîover 2 quintillion combinations
- **Video**: [Factorial Time Complexity - Jenny's Lectures](https://www.youtube.com/watch?v=ER8nVtdXl4s)

## Sorting Algorithms

### Bubble Sort
- **Definition**: Repeatedly swaps adjacent elements if they're in wrong order until the list is sorted
- **Analogy**: Bubbles rising in soda‚Äîlarger bubbles gradually float to the top
- **Video**: [Bubble Sort Algorithm - CS50](https://www.youtube.com/watch?v=RT-hUXUWQ2I)

### Selection Sort
- **Definition**: Finds the minimum element and places it at the beginning, repeating for remaining elements
- **Analogy**: Picking the smallest toy from a pile and setting it aside, then repeating
- **Video**: [Selection Sort - Gayle Laakmann McDowell](https://www.youtube.com/watch?v=g-PGLbMth_g)

### Insertion Sort
- **Definition**: Builds sorted array one element at a time by inserting each element into its correct position
- **Analogy**: Sorting playing cards in your hand‚Äîpick up each card and insert it where it belongs
- **Video**: [Insertion Sort Algorithm - mycodeschool](https://www.youtube.com/watch?v=i-SKeOcBwko)

### Heap Sort
- **Definition**: Converts array into a heap structure, then repeatedly extracts the maximum element
- **Analogy**: Priority queue at the ER‚Äîalways serve the most critical patient next
- **Video**: [Heap Sort - Abdul Bari](https://www.youtube.com/watch?v=MtQL_ll5KhQ)

### Quick Sort
- **Definition**: Divides array using a pivot element, recursively sorting elements smaller and larger than the pivot
- **Analogy**: Organizing books by picking a middle book and putting shorter books left, taller books right
- **Video**: [Quick Sort - CS50](https://www.youtube.com/watch?v=8hEyhs3OV1w)

### Merge Sort
- **Definition**: Recursively divides array in half until single elements remain, then merges sorted subarrays
- **Analogy**: Sorting two decks of cards by repeatedly picking the smaller top card from either deck
- **Video**: [Merge Sort Algorithm - mycodeschool](https://www.youtube.com/watch?v=TzeBrDU-JaY)

## Searching Algorithms

### Linear Search

- **Analogy**: Looking for your keys by checking every drawer one by one
- **Video**: [Linear Search - CS Dojo](https://www.youtube.com/watch?v=sGvcbgx_4q8)

## Graph Algorithms

### BFS (Breadth-First Search)
- **Definition**: Explores graph level by level, visiting all neighbors before moving to next depth
- **Analogy**: Ripple in a pond‚Äîexpands outward in concentric circles
- **Video**: [BFS Algorithm - William Fiset](https://www.youtube.com/watch?v=oDqjPvD54Ss)

### DFS (Depth-First Search)
- **Definition**: Explores graph by going as deep as possible before backtracking
- **Analogy**: Maze exploration‚Äîfollow one path to the end before trying another route
- **Video**: [DFS Algorithm - William Fiset](https://www.youtube.com/watch?v=7fujbpJ0LB4)

### Dijkstra's Algorithm
- **Definition**: Finds shortest weighted path in graph with non-negative edge weights
- **Analogy**: GPS navigation‚Äîfinds quickest route considering traffic (weights)
- **Video**: [Dijkstra's Algorithm - Computerphile](https://www.youtube.com/watch?v=GazC3A4OQTE)

### Bellman-Ford Algorithm
- **Definition**: Finds shortest path even with negative edge weights, detecting negative cycles
- **Analogy**: Navigation with toll roads where some give refunds (negative weights)
- **Video**: [Bellman Ford Algorithm - Abdul Bari](https://www.youtube.com/watch?v=FtN3BYH2Zes)

### A* Algorithm
- **Definition**: Pathfinding algorithm using heuristic function to estimate distance to goal
- **Analogy**: Treasure hunt with a compass‚Äîalways move toward where you think the treasure is
- **Video**: [A* Pathfinding - Sebastian Lague](https://www.youtube.com/watch?v=-L-WgKMFuhE)

## Greedy Algorithms

### Huffman Coding
- **Definition**: Compression algorithm assigning shorter codes to frequent characters
- **Analogy**: Shorthand writing‚Äîcommon words get abbreviations (Mr. for Mister)
- **Video**: [Huffman Coding - Tom Scott](https://www.youtube.com/watch?v=JsTptu56GM8)

### Prim's Algorithm
- **Definition**: Builds Minimum Spanning Tree by growing from a starting vertex
- **Analogy**: Building a subway system by always adding the cheapest connection to existing network
- **Video**: [Prim's Algorithm - Michael Sambol](https://www.youtube.com/watch?v=cplfcGZmX7I)

### Kruskal's Algorithm
- **Definition**: Builds Minimum Spanning Tree by sorting all edges and adding cheapest non-cycle-forming edges
- **Analogy**: Building roads by always choosing the cheapest option that doesn't create a loop
- **Video**: [Kruskal's Algorithm - Abdul Bari](https://www.youtube.com/watch?v=4ZlRH0eK-qQ)

### Ford-Fulkerson Algorithm
- **Definition**: Computes maximum flow through a network by finding augmenting paths
- **Analogy**: Water pipes‚Äîfind how much water can flow from source to sink through various pipes
- **Video**: [Max Flow Ford Fulkerson - William Fiset](https://www.youtube.com/watch?v=LdOnanfc5TM)

## Recursion

### Tail Recursion
- **Definition**: Recursive function where the recursive call is the last operation (can be optimized)

## Cache Replacement Policies

### LRU (Least Recently Used)
- **Definition**: Evicts the item that hasn't been accessed for the longest time
- **Analogy**: Cleaning out your closet‚Äîdonate clothes you haven't worn in years

### LFU (Least Frequently Used)
- **Definition**: Evicts the item accessed the fewest number of times
- **Analogy**: Removing apps you barely use from your phone
- **Video**: [LFU Cache Design - Tushar Roy](https://www.youtube.com/watch?v=0PSB9y8ehbk)

### MFU (Most Frequently Used)
- **Definition**: Evicts the item accessed most frequently (assumes popular items won't be needed soon)
- **Analogy**: Cycling out bestsellers at a library to make room for new books
- **Video**: [Cache Replacement Policies - Gate Lectures](https://www.youtube.com/watch?v=6LsyE7hI4oQ)

## String Search & Manipulation

- **Analogy**: Speed reading‚Äîskip paragraphs that can't possibly contain your search term
- **Video**: [Boyer Moore Algorithm - Back To Back SWE](https://www.youtube.com/watch?v=lkL6RkQvpMM)

## Complexity Classes

### P (Polynomial Time)
- **Definition**: Problems solvable in polynomial time (efficient algorithms exist)
- **Analogy**: Easy homework‚Äîyou can finish it in reasonable time
- **Video**: [P vs NP - Computerphile](https://www.youtube.com/watch?v=YX40hbAHx3s)

### NP (Nondeterministic Polynomial)
- **Definition**: Problems where solution can be verified quickly, but finding it may be hard
- **Analogy**: Sudoku‚Äîeasy to check if complete, harder to solve from scratch
- **Video**: [NP Completeness - MIT OpenCourseWare](https://www.youtube.com/watch?v=mr1FMrwi6Ew)

### NP-Hard
- **Definition**: At least as hard as the hardest NP problems
- **Analogy**: PhD-level exam questions‚Äîincredibly difficult
- **Video**: [NP-Hard vs NP-Complete - Easy Theory](https://www.youtube.com/watch?v=e2cF8a5aAhE)

### NP-Complete
- **Definition**: Problems that are both in NP and NP-Hard (hardest problems in NP)
- **Analogy**: Traveling salesman‚Äîeasy to check a route, hard to find the best one
- **Video**: [NP-Complete Problems - Gate Smashers](https://www.youtube.com/watch?v=e2cF8a5aAhE)

### Co-NP
- **Definition**: Complement of NP problems (easy to verify "no" answers)
- **Analogy**: Proving something is NOT a prime number (one counterexample suffices)
- **Video**: [Complexity Classes - CS Theory](https://www.youtube.com/watch?v=7d8s6KF4R80)

## Databases

### SQL
- **Definition**: Structured Query Language for relational databases with defined schemas
- **Analogy**: Organized library with card catalog‚Äîeverything has its place
- **Video**: [SQL Explained - Fireship](https://www.youtube.com/watch?v=zsjvFFKOm3c)

### NoSQL
- **Definition**: Non-relational databases offering flexible schemas (document, key-value, graph, etc.)
- **Analogy**: Flexible storage bins‚Äîthrow things in different containers as needed
- **Video**: [NoSQL Explained - Fireship](https://www.youtube.com/watch?v=0buKQHokLK8)

### Normalization
- **Definition**: Organizing database to reduce redundancy by splitting into related tables
- **Analogy**: Organizing kitchen‚Äîone drawer for utensils, one for dishes, not mixed
- **Video**: [Database Normalization - Decomplexify](https://www.youtube.com/watch?v=J-drts33N8g)

### Denormalization
- **Definition**: Adding redundancy to database for faster queries at cost of storage/consistency
- **Analogy**: Keeping snacks in multiple rooms‚Äîduplicates, but convenient
- **Video**: [Normalization vs Denormalization - Hussein Nasser](https://www.youtube.com/watch?v=lPT22lQ1ATA)

### Database Indexes
- **Definition**: Data structure that improves query speed at cost of write performance and storage
- **Analogy**: Book index‚Äîjump directly to topics instead of reading everything
- **Video**: [Database Indexing - CS Dojo](https://www.youtube.com/watch?v=fsG1XaZEa78)

### ACID (Transactions)
- **Definition**: Atomicity, Consistency, Isolation, Durability‚Äîguarantees for database transactions
- **Analogy**: Bank transfer‚Äîeither both accounts update or neither does
- **Video**: [ACID Properties - Simply Explained](https://www.youtube.com/watch?v=pomxJOFVcQs)

### BASE (Transactions)
- **Definition**: Basically Available, Soft state, Eventually consistent‚ÄîNoSQL alternative to ACID
- **Analogy**: Social media likes‚Äîmight take time to sync across servers
- **Video**: [CAP Theorem & BASE - Gaurav Sen](https://www.youtube.com/watch?v=k-Yaq8AHlFA)

### Replication
- **Definition**: Copying database to multiple servers for redundancy and read scalability
- **Analogy**: Photocopying important documents‚Äîbackups if one is lost
- **Video**: [Database Replication - Hussein Nasser](https://www.youtube.com/watch?v=bI8Ry6GhMSE)

### Sharding
- **Definition**: Horizontally partitioning data across multiple database servers
- **Analogy**: Splitting a phone book alphabetically across multiple volumes
- **Video**: [Database Sharding - Gaurav Sen](https://www.youtube.com/watch?v=5faMjKuB9bc)
